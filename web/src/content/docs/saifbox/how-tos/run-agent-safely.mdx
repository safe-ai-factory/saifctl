# Run an agent safely in Saifbox

This guide shows how to start an agent CLI inside a Saifbox container so the agent cannot reach your host filesystem, environment variables, or credentials.

## Prerequisites

- Docker installed and running
- `saifctl` installed (`npm install -g saifctl` or from the GitHub release)
- An agent CLI installed (for example, OpenClaw)

Saifbox mounts a copy of your project into the container. The agent sees that copy, not your real files.

**3. Confirm the run completes**

The terminal streams the agent's output. When the run ends, the container is destroyed. Your working tree is unchanged.

## What happens if the agent tries to write files

All writes go into the container filesystem. Nothing is applied to your project unless you pass `--extract` (see [Apply agent changes](/product/saifbox/how-tos/apply-agent-changes)).
