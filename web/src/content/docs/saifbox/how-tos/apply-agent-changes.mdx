# Apply agent changes to your working tree

By default, Saifbox discards everything the agent wrote when the container exits. This guide explains how to selectively extract the agent's git changes back to your host.

## Prerequisites

A completed (or paused) Saifbox run. See [Run an agent safely](/product/saifbox/how-tos/run-agent-safely).

## Steps

**1. Re-run with `--extract`**

After the agent finishes, Saifbox reads the git diff inside the container and applies it to your real working tree as unstaged changes.

The applied patch is identical to what the agent committed inside the container. Nothing is staged or committed on your behalf.
