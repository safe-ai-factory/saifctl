# saifctl sandbox

Run an agent inside an isolated Docker container without the `feat run` pipeline (no gate, reviewer, or staging tests). Use for one-off tasks, ad hoc experimentation, and setting up agents manually.

Nothing is written to your machine by default. Use `--extract` to apply the agent's changes to your working tree.

### Interactive (`--interactive` / `-i`)

Start the container, run startup and agent-install scripts, then drop you into a bash shell inside the container â€” no task, no agent loop:

<DocCodeBlock lang="bash" code="c2FpZmN0bCBzYW5kYm94IC0taW50ZXJhY3RpdmU=" />

You land directly in the container shell. When you type `exit` or press Ctrl+D, saifctl stops the container and (if `--extract` was passed) extracts any changes.

Use interactive mode when you need to configure the agent manually before it runs â€” for example, registering Claude MCP servers with `claude mcp add â€¦`.

## Arguments

### Required (non-interactive mode)

Exactly one of the following must be set unless `--interactive` is also set:

| Argument | Alias | Type | Description |
|---|---|---|---|
| `--task` | `-t` | string | Task prompt for the agent. |
| `--task-file` | â€” | string | Path to a file whose contents become the task prompt (relative to `--project-dir` if not absolute). |
| `--subtasks` | â€” | string | Path to a subtasks JSON manifest (same format as [`feat run --subtasks`](/product/saifctl/references/commands/feat-run#subtasks)). Cannot be combined with `--task` or `--task-file`. |

### Mode

| Argument | Alias | Type | Description |
|---|---|---|---|
| `--interactive` | `-i` | boolean | Start an interactive sandbox: runs startup and agent-install scripts, then idles until Ctrl+C. Mutually exclusive with `--task`, `--task-file`, and `--subtasks`. |

### Output

| Argument | Alias | Type | Description |
|---|---|---|---|
| `--extract` | â€” | boolean | After the run (or session), apply the agent's git changes to the host working tree as unstaged changes. Works in both interactive and non-interactive modes. |
| `--extract-include` | â€” | string | Repo-relative path prefix: only extract changes under this path. Requires `--extract`. |
| `--extract-exclude` | â€” | string | Repo-relative path prefix: exclude from the extracted patch. Requires `--extract-include`. |

### Identity

| Argument | Alias | Type | Description |
|---|---|---|---|
| `--name` | `-n` | string | Label for this sandbox run (kebab-case). Default: random `scratch-*`. |

### Environment & agent

These follow the same semantics as [`feat run`](/product/saifctl/references/commands/feat-run). See that page for full descriptions.

| Argument | Description |
|---|---|
| `--agent` | Agent profile (default: `openhands`). |
| `--agent-script` | Path to the agent run script. Overrides profile default. |
| `--agent-install-script` | Path to the one-time agent install script. |
| `--startup-script` | Shell script run once to install workspace dependencies. |
| `--profile` | Sandbox profile (default: `node-pnpm-python`). |
| `--coder-image` | Docker image for the coder container. |
| `--agent-env` | Extra env vars for the agent container (`KEY=VALUE`). Repeatable. |
| `--agent-env-file` | Path(s) to `.env` file(s) forwarded into the container. |
| `--model` | LLM model override. |
| `--project-dir` | Project directory (default: current directory). |
| `--saifctl-dir` | Path to saifctl directory (default: `saifctl`). |
| `--engine` | Engine override: `docker`, `local`, or `helm`. |
| `--cedar` | Absolute path to Cedar policy file for Leash. |
| `--include-dirty` | Include uncommitted and untracked files in the sandbox. |
| `--verbose` | Enable verbose logging. |

## Validation

| Condition | Error |
|---|---|
| Neither `--task`, `--task-file`, `--subtasks`, nor `--interactive` provided | `Error: provide --task/-t, --task-file, --subtasks, or --interactive` |
| `--interactive` combined with `--task`, `--task-file`, or `--subtasks` | `Error: --interactive cannot be combined with --task, --task-file, or --subtasks` |
| `--task` and `--task-file` both set | `Error: use either --task or --task-file, not both` |
| `--subtasks` combined with `--task` or `--task-file` | `Error: do not combine --subtasks with --task or --task-file` |
| `--extract-exclude` without `--extract-include` | `Error: --extract-exclude requires --extract-include` |
| `--extract-include` without `--extract` | `Error: --extract-include requires --extract` |
| `--name` not kebab-case | `Error: --name must be kebab-case â€¦` |

## What this command skips compared to `feat run`

| Capability | `feat run` | `sandbox` |
|---|---|---|
| Gate script | Yes | No (noop gate) |
| Reviewer | Yes | No |
| Staging tests | Yes | No |
| Multiple outer attempts | Yes (`--max-runs`) | No (always `maxRuns=1`) |
| Run storage | Yes | No (`storage=none` by default) |
