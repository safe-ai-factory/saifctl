# saifctl doctor

Diagnose environment problems before running `feat run` or `sandbox`. Checks Docker, the Leash CLI, and optional Hatchet connectivity in sequence.

## Usage

<DocCodeBlock lang="bash" code="c2FpZmN0bCBkb2N0b3I=" />

No arguments or flags.

## Checks

Runs three checks in order. Any hard failure exits with code `1` and prints an error. All must pass for the final "All checks passed." message.

| # | Check | Pass condition | Failure condition |
|---|-------|---------------|-------------------|
| 1 | **Docker** | `docker info` succeeds | Docker not running or not reachable → hard failure |
| 2 | **Leash CLI** | Binary resolved and exists on disk | Path not found or binary missing → hard failure |
| 3 | **Hatchet** | Token absent (local mode, warning only) **or** token set and client initialises in server mode | Token set but SDK throws, or client initialises in local mode → hard failure |

### Hatchet check detail

- **`HATCHET_CLIENT_TOKEN` unset** — prints a warning and continues. `saifctl` operates in local (in-process) mode. Not an error.
- **`HATCHET_CLIENT_TOKEN` set** — initialises the Hatchet client. If the client reports local mode, the check fails. Otherwise success is logged with the resolved server address.

## Environment variables

| Variable | Required | Default | Description |
|---|---|---|---|
| `HATCHET_CLIENT_TOKEN` | No | — | Hatchet auth token. If unset, saifctl runs in local mode. If set, doctor tests client initialisation. |
| `HATCHET_SERVER_URL` | No | `localhost:7077` | gRPC address shown in success output when the Hatchet check passes. |

## Exit codes

| Code | Meaning |
|------|---------|
| `0` | All checks passed (including the case where `HATCHET_CLIENT_TOKEN` is unset). |
| `1` | One or more hard checks failed (Docker down, Leash CLI missing, or Hatchet client error). |
